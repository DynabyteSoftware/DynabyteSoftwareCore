49  * This Exception class is meant to provide additional information that std::exception does not (it does inherit from
50  * std::exception, so it can be handled normally)
51  *
52  * The additional information it contains is the function name, source filename, line number, and potentially
53  * inner exceptions in the case of cascaded exception-handling. If thrown using the intended THROW macro, this
54  * information is populated automatically
55  *
56  * In addition, this class also automatically self-logs whenever it is thrown, using std::clog. If Log has been
57  * initialized with a Log::Error logfile, clog will output to the selected file stream.
58  *
59  * @note
60  * When inheriting from this class, ensure the constructor parameters start by taking the function name,
61  * the source filename, and the line number--in this order. Additional parameters may be added as desired, after
62  * the required parameters. This setup ensures your exceptions will be able to use the THROW macro to automatically
63  * populate that information
64  ********************************************************************************************************************/
