46  * @details
47  * This Exception class is meant to provide additional information that std::exception does not (it does inherit from
48  * std::exception, so it can be handled normally)
49  *
50  * The additional information it contains is the function name, source filename, line number, and potentially
51  * inner exceptions in the case of cascaded exception-handling. If thrown using the intended THROW macro, this
52  * information is populated automatically
53  *
54  * In addition, this class also automatically self-logs whenever it is thrown, using std::clog. If Log has been
55  * initialized with a Log::Error logfile, clog will output to the selected file stream.
56  *
57  * @note
58  * When inheriting from this class, ensure the constructor parameters start by taking the function name,
59  * the source filename, and the line number--in this order. Additional parameters may be added as desired, after
60  * the required parameters. This setup ensures your exceptions will be able to use the THROW macro to automatically
61  * populate that information
62  ********************************************************************************************************************/
